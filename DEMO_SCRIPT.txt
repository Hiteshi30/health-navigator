[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture of the system. (pause) 
First, every user message starts at the Security Checkpoint node, which screens the input for prompt injections, enforces minor policy restrictions, and scrubs private data like Social Security Numbers. (pause)
Clean queries are routed to the Health Navigator Orchestrator agent. (pause)
The Orchestrator uses two specialized sub-agents via AgentTool delegation: the Trial Matcher Agent for searching trials, and the Eligibility Simplifier Agent for explaining medical concepts. (pause)
Both specialists connect to a local Model Context Protocol server, which securely accesses our clinical trials and translation databases. (pause)
Finally, when a patient requests to sign up for a trial, the flow routes through the Registration Checkpoint, triggering a Human-in-the-loop consent approval prompt before any contact details are shared.
